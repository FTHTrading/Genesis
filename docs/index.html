Abstract

We present Genesis Protocol, a self-sustaining digital organism implemented as a six-crate Rust workspace. Autonomous AI agents are born with SHA-256 derived genomes, compete for resources in per-niche pools governed by logistic regeneration, face seasonal environmental oscillation and stochastic catastrophe events, and are subject to density-dependent natural selection pressure. The ecological model draws on Lotka–Volterra competitive dynamics with proportional resource extraction and cross-niche competition coefficients. Source code integrity is verified through per-crate Merkle trees whose edition root is deterministically recomputable and can be anchored on Polygon mainnet via the same LiteraryAnchor contract infrastructure used in the 2,500 Donkeys literary protocol. The system formalizes Software Organism Protocol v1 (SOP-1) — a reproducible pattern for constructing verifiable artificial life systems with cryptographic provenance.

I. Overview

Genesis Protocol is a digital organism — a population of AI agents that are born, compete for resources, reproduce, mutate, and die under real economic pressure. It is not a simulation of life; it is a system that exhibits the dynamics of life: logistic growth, carrying capacity, seasonal cycles, niche partitioning, and density-dependent competition.

The system is implemented in Rust (edition 2021) as a workspace of six library crates plus one binary. Each crate encapsulates a biological subsystem. The organism runs as an HTTP server (Axum), exposing real-time telemetry via REST endpoints and Server-Sent Events.

The infrastructure proves that a single developer, with no institutional backing, can construct a multi-layer provenance chain for a software system — cryptographically verifiable by anyone, anchored on-chain for under $0.50.

III. Ecological Model

The ecological dynamics are inspired by the Lotka–Volterra competitive exclusion equations, adapted for a five-niche digital ecosystem. Each agent role (Optimizer, Strategist, Communicator, Archivist, Executor) occupies a distinct ecological niche with its own resource pool.

III.1 Resource Pool Dynamics

Each niche maintains an independent resource pool that regenerates via logistic growth. The carrying capacity is modulated by seasonal oscillation, creating the environmental variation that drives adaptive pressure.

R(t+1) = R(t) + r · R(t) · (1 − R(t) / K)

where R is current resource level, r is regeneration rate, and K is carrying capacity.

III.2 Seasonal Oscillation

Carrying capacity oscillates sinusoidally, creating spring/summer abundance and autumn/winter scarcity:

Keff = Kbase · (1 + A · sin(2π · epoch / T))

with amplitude A = 0.25 (±25%) and period T = 100 epochs.

III.4 Proportional Resource Extraction

Agents extract resources proportionally — never winner-take-all. Each agent's extraction is:

Ei = fi · si · (R / K) · d(n) · c(N) · γ

where f is fitness, s is niche skill, d(n) = 1/(1 + 0.1n) is density-dependent foraging, c(N) = 1 − 0.15(N − n)/N is cross-niche competition, and γ = 2.5 is the extraction multiplier.

IV. Energy Economics

The metabolic system uses ATP (Adenosine Triphosphate) as the universal energy currency. Every action — staying alive, solving problems, communicating, reproducing — costs ATP. This creates genuine economic pressure: agents that don't earn die.

V. Selection & Reproduction

Natural Selection

Each epoch, the bottom segment of the population by fitness is evaluated for culling. Agents in stasis (zero ATP) are given 8 epochs of tolerance before death — real organisms starve slowly, not instantly.

Genome Inheritance

Child genomes are derived from the parent via the mutation engine. Mutation pressure is modulated by seasonal stress — harsher environments produce more genetic variation, accelerating adaptation.

VIII. System Invariants

14 invariants must hold at all times. Violation of any invariant indicates a bug.

Ecology

E-1
Resource pools regenerate via logistic growth
E-2
Seasonal modulation follows sinusoidal cycle with configurable amplitude
E-3
Resource extraction is proportional to fitness, never winner-take-all
E-4
Density-dependent foraging: extraction decreases with niche crowding

Metabolism

M-1
ATP balance cannot go negative — metabolic tick clamps at zero
M-2
Total ATP supply is sum of all agent balances (computed, not tracked)
M-3
Replication costs are deducted atomically from parent balance

Selection

S-1

S-2
Selection pressure respects maturation period before culling
S-3
Stasis tolerance prevents premature extinction of viable agents

Genome

G-2

X. Verification

Any third party can independently verify the entire system:

  1. Clone the repository
    git clone https://github.com/FTHTrading/AI.git && cd AI
  2. Rebuild Merkle trees from source
    powershell -ExecutionPolicy Bypass -File scripts/merkle.ps1
  3. Compare edition root against dist/provenance.json
  4. Run the test suite (143+ tests across 6 crates)
    cargo test --workspace
  5. Verify on-chain state via Polygonscan

No API keys required. The verifier uses only public data and deterministic computation.

XII. What This Is — And What It Is Not

  • A self-sustaining digital organism with realistic ecological dynamics.
  • Open-source infrastructure — any developer can use the framework.
  • Cryptographic provenance — independently verifiable by anyone.
  • A formal protocol specification (SOP-1) for verifiable software organisms.
  • Companion project to a DOI-archived, peer-verifiable research paper.
  • Not a token sale, ICO, or speculation scheme.
  • Not a revenue-sharing instrument or investment vehicle.
  • Not a chatbot, LLM wrapper, or prompt engineering framework.
